# Contributing to The Mission Kitchen

Thank you for your interest in contributing to The Mission Kitchen website! This document provides guidelines and instructions for contributing.

## Code of Conduct

- Be respectful and inclusive
- Welcome newcomers and help them get started
- Focus on what is best for the community
- Show empathy towards other community members

## How to Contribute

### Reporting Bugs

If you find a bug, please create an issue with:
- Clear description of the problem
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if applicable)
- Browser and OS information

### Suggesting Enhancements

We welcome suggestions! Please create an issue with:
- Clear description of the enhancement
- Why it would be useful
- Examples of how it would work
- Mockups or designs (if applicable)

### Pull Requests

1. **Fork the repository**
   ```bash
   git clone https://github.com/yourusername/mission-kitchen-website.git
   ```

2. **Create a branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Make your changes**
   - Write clean, readable code
   - Follow existing code style
   - Test thoroughly in multiple browsers

4. **Commit your changes**
   ```bash
   git add .
   git commit -m "Add: Brief description of changes"
   ```

5. **Push to your fork**
   ```bash
   git push origin feature/your-feature-name
   ```

6. **Create a Pull Request**
   - Provide clear description of changes
   - Reference any related issues
   - Include screenshots of UI changes

## Development Guidelines

### Code Style

- Use consistent indentation (2 spaces)
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions small and focused

### Testing

Before submitting, test your changes:
- Desktop browsers (Chrome, Firefox, Safari, Edge)
- Mobile browsers (iOS Safari, Chrome Mobile)
- Different screen sizes
- All interactive features

### Commit Messages

Use clear commit messages:
- `Add: New feature or file`
- `Fix: Bug fix`
- `Update: Changes to existing feature`
- `Refactor: Code improvements`
- `Docs: Documentation changes`
- `Style: CSS/styling changes`

## Feature Ideas

Some areas where contributions would be welcome:
- Additional meal plans
- More recipe examples
- Animation improvements
- Accessibility enhancements
- Performance optimizations
- Mobile experience improvements
- New sections (testimonials, blog, etc.)
- Internationalization/localization

## Questions?

Feel free to open an issue for any questions about contributing.
